# AI 기반 다중 가이드라인 보안 점검 플랫폼

> Linux/Windows 시스템에서 사실 정보를 수집하고, 선택한 보안 가이드라인 또는 업로드한 PDF 기준 문서를 바탕으로 RAG + LLM이 판정하는 프로젝트

## 프로젝트 한 줄 소개

이 프로젝트는 단순한 `주통기 전용 점검기`가 아니라, 여러 보안 가이드라인을 선택해 적용할 수 있고 회사별 기준 PDF까지 받아서 판정에 활용할 수 있는 `다중 가이드라인 기반 보안 점검 플랫폼`을 목표로 합니다.

현재 방향의 핵심은 다음과 같습니다.

- 수집 스크립트는 `양호/취약`을 판단하지 않고 사실 정보만 수집
- 최종 판정은 `RAG + LLM`이 수행
- 기본 DB는 지금 바로 확인하기 위해 `SQLite`로 테스트 중
- 최종 운영 DB는 아직 미정이며, 구조는 교체 가능하게 유지
- LLM은 `Gemini API 2.5` 기반으로 전환 예정
- 웹에서 가이드라인을 선택하고, 필요하면 회사 내부 PDF를 업로드해서 새 기준으로 활용 예정

## 현재 팀 방향

### 1. LLM 호출 방식 변경

- 기존 CLI 중심 흐름에서 `Gemini API 2.5` 기반으로 전환 예정
- 사용량 제한을 고려해 `분당 15회`, `하루 1500회` 수준의 rate limit 관리가 필요
- 따라서 `작업 큐`, `재시도`, `호출 상태 기록`, `실패 복구` 구조가 중요해짐

### 2. 웹 서비스화

- 도메인을 연결한 웹 서비스 형태로 운영 예정
- 팀원 노트북을 서버로 사용해 시연/운영할 계획
- FastAPI 기반 웹에서 수집 결과 업로드, 판정 실행, 결과 조회, 보고서 다운로드를 제공

### 3. 가이드라인 선택 기능

- 주통기만이 아니라 여러 기준 문서를 넣고 싶어 하는 요구를 반영
- 사용자가 웹에서 가이드라인을 선택하면, RAG 검색 범위도 그 기준으로 제한
- 즉, 앞으로는 `scan -> selected guideline -> retrieval -> judgment` 흐름이 됨

### 4. 회사별 PDF 업로드

- 각 회사 내부 보안 기준서나 점검 기준 PDF를 업로드해서 판정에 활용하는 기능을 목표로 함
- 업로드된 문서도 기존 가이드라인과 같은 방식으로 `파싱 -> 청킹 -> 임베딩 -> 검색` 흐름에 들어감
- 장기적으로는 `기본 가이드라인`과 `사용자 업로드 문서`를 같은 RAG 구조 안에서 다룰 예정

## 핵심 아이디어

### 수집과 판정을 분리한다

이 프로젝트에서 수집 스크립트의 역할은 다음으로 제한합니다.

- 실제 시스템 설정값 수집
- 원본 명령어 출력 저장
- 판정에 필요한 근거 정보 구조화

판정은 LLM이 수행합니다.

- 선택한 가이드라인을 기준으로 관련 문서 검색
- 수집 결과와 기준 문서 비교
- `양호`, `취약`, `판정불가`와 같은 결과 생성
- 이유와 조치 방안 설명

즉, `collector = 사실 수집`, `LLM = 해석과 판단` 구조입니다.

## 목표 시스템 구조

```text
사용자
  -> 웹에서 점검 결과 업로드 또는 수집 실행
  -> 적용할 가이드라인 선택
  -> 필요시 회사 기준 PDF 업로드

수집 계층
  -> Linux / Windows collector
  -> 사실 정보(JSON) 생성

지식 계층
  -> 기본 가이드라인 PDF
  -> 사용자 업로드 PDF
  -> 파싱 / 청킹 / 임베딩 / 벡터 저장

판정 계층
  -> 선택된 guideline_id 범위로 RAG 검색
  -> Gemini API 2.5로 항목별 판정
  -> rate limit / 재시도 / 결과 저장

저장 / 결과 계층
  -> DB 저장
  -> 판정 결과 조회
  -> 보고서 생성
```

## 현재 기준 기능 요약

### 이미 반영된 방향

- Windows collector 결과를 구조화해서 저장
- 수집 결과에 `제목`, `세부 수집 결과`, `명령어`, `주통기 매핑` 같은 정보 포함
- 기본 실행 시 DB 저장 시도 가능
- 개발/확인용으로 SQLite 사용 가능

### 앞으로 더 커질 부분

- 다중 가이드라인 관리
- 가이드라인 선택 UI
- 회사별 PDF 업로드
- 업로드 문서 기반 RAG
- Gemini API 호출 제한 관리
- 운영용 배포 구성
