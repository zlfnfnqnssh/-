# AI 기반 취약점 점검 및 자가 조치 웹 플랫폼 - 핵심 워크플로우 명세

## 1. 플랫폼 워크플로우 전체 흐름

이 플랫폼은 **[관리자(Admin) 환경]**과 **[사용자(User) 환경]**으로 나뉘며, 웹 기반 인터페이스를 통해 시스템 보안 점검과 가이드라인 업데이트 스크립트 생성 과정을 통합 관리합니다.

### 1-1. 관리자(Admin) - 지식 및 점검 스크립트 갱신 자동화
1. **주통기 가이드라인 업데이트:** 최신 주통기(주요정보통신기반시설) PDF 문서 배포 시, 관리자 페이지에서 웹 업로드.
2. **파싱 및 비교:** 문서를 PyMuPDF로 파싱하고 DB에 저장된 이전 버전의 가이드라인과 비교하여 추가/수정/삭제 내역을 도출.
3. **스크립트 자동화 (Gemini CLI):** 비교 결과를 바탕으로, 새롭게 추가되거나 변경된 항목에 맞는 점검 스크립트 코드를 Gemini CLI가 생성.
4. **저장/동기화:** 생성된 스크립트를 OS별 폴더(`scripts/windows/`, `scripts/linux/`)에 저장.

### 1-2. 사용자(User) - 원클릭 점검 및 스마트 조치
1. **사용자 환경 로그인/식별:** UUID 기반 개인 DB 세션 및 진단 이력 관리.
2. **점검 스크립트 실행:** 웹에서 "점검 시작" 버튼 클릭 시, 시스템 환경(Linux/Windows) 자동 식별 후 해당 스크립트 즉시 실행.
3. **점검 데이터 수집:** 스크립트가 로컬 상태를 점검하여 JSON 형태로 데이터 생성.
4. **결과 처리 및 LLM 판단 (3단계 스마트 파이프라인):**
   - **1단계 분류:** 규칙 기반 결과(`양호`/`취약`/`규칙불가`)에 따라 항목을 분류
   - **2단계 LLM 호출:**
     - 규칙 기반 `양호`: Batch 일괄 LLM 검토 (이상 발견 시 개별 재처리)
     - `취약`/`규칙불가`: 개별 병렬 LLM 호출 (MAX_CONCURRENT=5 세마포어 제어)
   - **3단계 저장:** 모든 판정 결과 DB 저장
5. **LLM 반환 결과 (DB 저장):**
   - 취약점 원인에 대한 쉬운 설명 및 짧은 공격 시나리오
   - 주통기 가이드라인 기반 조치 방법
   - 즉시 실행 가능한 패치 스크립트 (Linux: bash, Windows: PowerShell)
   - 판정 신뢰도 (confidence score)
6. **후처리:** 분석 종료 후 임시 JSON 삭제, 결과를 DB에 반영하여 웹 대시보드에서 설명/조치 영역을 분리 표시.

### 1-3. 재점검 (시계열 비교)
- 사용자가 재점검 시 현재 결과를 이전 점검 이력과 자동 대조.
- "여전히 취약", "새롭게 취약", "양호로 전환" 등 상태 변화를 웹 UI에서 필터링 및 시각화.

## 2. 병렬 파이프라인 데이터 흐름 최적화 전략

### 점검 성능/비용 최적화 (규칙 기반 + LLM 분배 전략)
* **Rule-based 필터링 1차:** 스크립트 상에서 명확하게 상태를 식별할 수 있는 항목들은 로컬에서 자체 판단하여 양호/취약을 분류.
* **승인 목적 일괄 전송 (Batch):** 규칙 기반 "양호" 건은 JSON 하나로 패키징해 Gemini에게 넘겨 확정만 받음 (동시 API 호출 불필요). 이상 발견 시 해당 건만 개별 재처리.
* **단건 병렬 전송 (Parallel / Async):** 취약 건이나 AI 설명이 필요한 항목들은 여러 Task로 쪼개어 비동기 병렬로 Gemini CLI 호출, 응답 딜레이 최소화.

## 3. 웹 플랫폼 화면 구성

- **Login 페이지:** User / Admin 인증
- **Admin 대시보드:**
  - 주통기 버전 현황 및 히스토리
  - 새 주통기 PDF 업로더
  - 가이드라인 비교 뷰
  - 스크립트 재생성 버튼
- **Users 메인보드:**
  - 최근 점검 대시보드 (진단 스코어)
  - 원클릭 점검 시작 버튼 (OS 자동 판단)
- **Users 점검 결과 대시보드:**
  - 상태별 비교 (이전 vs 현재)
  - 항목별 결과 카드 (취약/양호 색상 구분)
  - 취약 카드 상세: [원인 설명] 탭 + [조치 가이드 및 패치 스크립트] 탭
  - 패치 스크립트 실행 버튼

## 4. 점검 항목 현황

| OS | 항목 범위 | 구현 스크립트 | 비고 |
|----|-----------|--------------|------|
| Linux (Unix) | U-01 ~ U-72 | 56개 | bash 기반 수집 |
| Windows | W-01 ~ W-47 | 32개 | PowerShell 기반 수집 |

모든 스크립트는 주통기 가이드라인 DB 항목코드/항목명과 1:1 매칭됩니다.
