## [6] 제안 시스템 — 3대 핵심

**한 줄 정의**
> 수집 스크립트 + 가이드라인 DB + LLM 판정을 결합한
> **"맥락 이해 · 조치 자동화 · 자가 갱신"** 3단 E2E 주통기 진단 플랫폼

### ① 맥락 이해 — 하이브리드 판정
- 규칙 1차(양호/취약) → **LLM 2차 재판정**
- 주통기 가이드라인 DB(`vs_guideline_items`)에서
  `criteria + remediation + check_examples` 를 **프롬프트에 직접 주입**
- → 주석 처리·조건부 설정 등 **오탐 감소**

### ② 조치 자동화 — 탐지에서 실행·재작성까지
- LLM이 `patch_script` 자동 생성 (PowerShell / Bash)
- **UAC / sudo** 자동 승격 실행
- 실패 시 `stderr + 가이드라인` → **Gemini 재호출로 스크립트 재작성**
- 패치 전·후 재수집으로 **자체 검증**

### ③ 자가 갱신 — 가이드라인 개정 자동 반영
- 신규 주통기 PDF 투입 → 파싱 → **기존 DB와 자동 diff** (add / mod / del)
- 변경분을 **Gemini CLI**로 전달하여 점검 스크립트 자동 생성·재작성·삭제
- → **매년 KISA 개정을 사람 손 없이 반영**

🖼️ *3대 핵심을 3컬럼 아이콘 다이어그램으로 시각화*

## [8] 기술 스택 & 선정 근거

| 영역 | 기술 | 선정 이유 |
|---|---|---|
| LLM | Gemini CLI | **API 키 불필요**, 비용 0, 로컬 subprocess |
| RDB | PostgreSQL | 가이드라인·스캔·판정 전체 저장, 비동기 지원 |
| 웹 | FastAPI + Jinja2 | Python 단일 스택, SSR로 단순 |
| 수집 | PowerShell / Bash | OS 네이티브 실행, 별도 에이전트 불필요 |
| 병렬화 | asyncio.Semaphore | 5개 동시 수집으로 1분+ → 수십 초 단축 |

🖼️ *각 기술 로고 나란히 (Python / FastAPI / PostgreSQL / Gemini CLI)*
