## 🪟 Windows 스크립트 82개 전체 재정비

### 주통기 기준 적용 범위
- **W-01 ~ W-64** (서버 점검, 64개)
- **PC-01 ~ PC-18** (PC 점검, 18개)
- **합계 82개**

### 판정 확정 결과
| 분류 | 개수 | 비율 |
|---|:-:|:-:|
| 🟢 양호 확정 | **43** | 52% |
| 🔴 취약 확정 | **26** | 32% |
| 🟡 규칙불가 (LLM 판정) | **13** | 16% |

### Category B (LLM 판정 대상, 13개)
- W: W-03, W-06, W-18, W-27, W-35, W-37, W-38, W-40, W-62 (9개)
- PC: PC-05, PC-06, PC-08, PC-10, PC-11 (4개)

### 스크립트 공통 개선
- 모든 스크립트에 **주통기 원문 기준 + 파싱 결과 + 판정 가이드** 포함
- Category A: 직접 양호/취약 반환 + 근거 데이터
- Category B: `규칙불가` 유지 + LLM이 판정 가능한 풍부한 진단 정보
- Windows 한글 인코딩 (UTF-16LE/BE, UTF-8, CP949) 자동 감지

## 🔧 코드 개선 6건 (백엔드)

### 1. Gemini CLI 프롬프트에서 `confidence` 출력 지시 제거
- `engine/llm_judge.py` 4곳 수정
- `_build_prompt`, `_build_batch_prompt`, `_build_prompt_v2`, `_build_batch_prompt_v2`

### 2. Gemini CLI 로그 간소화
- Before: `cmd=npx @google/gemini-cli -m ... -p "..."` 전문 + stdout 2000자 덤프
- After: `[LLM] PC-08 판정 시작 (1984자)` / `[LLM] PC-08 완료 (2.3s, 312자, 취약)`

### 3. machine_id 기반 scan_id 체계 (신규)
- `config/machine_id.py` 신규 생성
- Windows: `wmic csproduct get UUID` / Linux: `/etc/machine-id`
- fallback: MAC + hostname SHA256 해시
- scan_id 형식: `{os}_{machine_id}_{user_short}_{timestamp}`
- 같은 PC·사용자·OS는 prefix 공유 → 이력 조회 가능

### 4. 웹 대시보드 진행률 % 개편
- `web/routes/scan.py` — `/scan/progress/{scan_id}`
- 수집 0~30% + LLM 판정 30~100% 통합
- 응답에 `overall`, `phase_label` 추가

### 5. PDF 리포트 자동 생성
- 스캔 완료 후(`comparing` 직후) `generate_report()` 자동 호출
- 실패해도 비치명적 처리 (스캔 자체는 완료 유지)

### 6. UAC 패치 실행 + Gemini 재작성 루프
- `web/routes/patch.py` 전면 재작성
- Windows: `Start-Process -Verb RunAs` + Base64 인코딩 + 로그 파일 회수
- Linux: sudo 실행
- 실패 시 Gemini에 (현재 스크립트 + 오류 + criteria + remediation + check_examples) 전달해 재작성
- 최대 3회 재시도, 성공한 스크립트로 DB `patch_script` 갱신
